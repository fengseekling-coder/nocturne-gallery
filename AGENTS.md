# Gega Gallery — 统一项目规范

> 版本：v5.5
> 更新日期：2026-04-21

本文件是所有 AI Agent 共享的项目规范。任何 Agent 在本项目中实现功能、修复问题、更新样式、修改数据流时，都必须遵守本文件。

---

## 1. 项目概述

**Gega Gallery（灵感库）** 是一款纯本地桌面端素材管理软件。

- 用户群体：设计师、创意工作者
- 核心能力：素材管理、分组隔离、搜索检索、AI 提示词管理、重复检测、拖拽导入导出
- 技术栈：Tauri 2 + React 18 + TypeScript + Tailwind CSS + SQLite + Rust
- 设计风格：Arc-Inspired Minimalist，深色沉浸式
- 界面语言：中文

## 3. 布局规范

### 3.1 三栏结构

- Sidebar：固定 `192px`
- Canvas：`flex-1`
- Inspector：默认 `256px`，可拖拽范围 `240px ~ 600px`

### 3.2 顶部对齐

以下三个区域高度必须严格一致，均为 `48px`：

- Sidebar Logo 区
- Canvas Topbar
- Inspector Topbar

并统一使用 `1px solid var(--border)` 作为底部分隔。

### 3.3 Sidebar 结构

Sidebar 必须拆成三段：

1. 顶部主导航区：固定
2. 中部自定义分组区：独立滚动
3. 底部设置区：固定

禁止把主导航和分组列表放在同一个压缩容器内。

### 3.4 分组显示逻辑

- 主导航按钮是“大分组”
- 自定义分组是“大分组”下的“小分组”
- 点击大分组：显示该大分组下全部素材，包含其子分组素材
- 点击小分组：只显示该分组内素材，不能显示其他分组素材
- 主导航与小分组在视觉上必须是单选态，不能同时高亮
- “AI 提示词库”是虚拟大分组，不对应物理文件夹；其列表和计数必须从全库按 Prompt / AI 元数据条件筛选，不能用 `source_folder = 'AI 提示词库'` 限制结果

## 4. 当前交互事实

以下行为已确定，后续开发必须保持一致：

### 4.1 Prompt 区

- Prompt 区允许直接编辑
- “查看完整”不是弹窗，而是在 Inspector 内展开高度
- 鼠标滚轮在 Prompt 文本区上滚动时，必须带动右侧面板正常滚动
- Prompt 下方有“附件区”，用于挂接源文件、工程文件、参考素材等外部附件

### 4.2 预览与属性面板

- 双击查看大图时，右侧属性面板必须同步
- 右键“查看大图”进入预览时，右侧属性面板也必须同步
- 预览翻页时，属性面板必须跟着切换，不能空白、错位或滞后

### 4.3 提示系统

- 内容区普通提示统一为“内容区底部居中的胶囊提示”
- 导入开始 / 导入中 / 导入完成统一在一个导入提示里完成
- 删除确认、重复素材确认等需要用户决策的交互必须使用居中弹窗

### 4.4 重复素材导入

- 重复素材不走底部 toast
- 必须弹出居中确认框
- 弹窗中必须显示已有素材所在的大分组 / 小分组
- 必须允许用户决定：
  - 继续导入
  - 使用已有素材

### 4.5 媒体卡片

- 卡片右上角收藏图标已移除，禁止恢复
- 选中态高亮不使用缓动动画，必须直接切换
- 卡片允许右键菜单、双击预览、拖拽导入导出

### 4.6 拖拽

已支持：

- 从本地文件拖入应用
- 从本地文件夹拖入应用并递归导入
- 从网页拖拽素材到应用
- 从应用内将单个素材拖出到外部
- 从应用内将多个已选素材一起拖出到外部

要求：

- 外部目标是否接受拖拽由对方决定，我们只负责正确输出拖拽数据
- 拖入与拖出都必须有明确反馈，不允许静默无响应

## 5. 代码规范

### 5.1 TypeScript

- 禁止 `any`
- 组件必须有明确 Props 类型
- 优先函数式组件
- 共享数据结构优先放在 `src/types`
- 字体必须统一走全局 `--font-family`，禁止在组件内随意指定其他字体

### 5.2 Rust / Tauri

- 所有 Command 必须有错误处理
- 对外错误统一返回可读信息
- 涉及扫描 / 导入 / 重建等耗时操作必须发送进度事件

### 5.3 组件命名

| 类型 | 规则 | 示例 |
|------|------|------|
| React 组件 | PascalCase | `MediaCard.tsx` |
| Hook | camelCase + `use` 前缀 | `useMediaStore.ts` |
| 类型定义 | camelCase | `media.ts` |
| Rust 模块 | snake_case | `thumbnail.rs` |

---

## 6. 数据与隔离原则

- 所有持久化数据走 SQLite / Rust Command
- 禁止 `localStorage`
- 禁止扫描库根目录之外的路径
- 禁止修改原始文件名
- 大分组可查看其下全部内容
- 小分组只能查看自身内容
- 重复素材检测优先基于内容而不是文件名

## 7. 已落地功能清单

### 7.1 素材与性能

- Masonry 瀑布流
- 图片多层缩略图加载
- 滚动时避免卡片闪烁 / 黑屏
- 视频首帧缩略图
- 重复检测：SHA256 + pHash
- Inspector 与预览联动

### 7.2 分组与导航

- 主导航固定 + 自定义分组独立滚动
- 分组单选高亮
- 大分组 / 小分组内容隔离

### 7.3 导入导出

- 本地文件拖入
- 文件夹拖入
- 网页拖入
- 单文件 / 多文件拖出到外部
- 导入进度统一提示

### 7.4 提示与弹窗

- 底部胶囊提示统一化
- 重复素材居中确认弹窗
- 删除确认弹窗

### 7.5 Inspector 附件区

- 附件区位于 Prompt 下方
- 附件为外部文件引用，不复制进素材库
- 支持多选添加、打开文件、打开所在位置、移除附件
- 附件默认使用两列瀑布式缩略图卡片显示
- 普通图片直接显示原图缩略图，设计源文件优先读取系统缩略图预览

## 8. 禁止事项

1. 禁止添加登录 / 注册
2. 禁止添加云同步 / 上传
3. 禁止硬编码颜色
4. 禁止 `any`
5. 禁止在组件中直接写 SQL
6. 禁止使用系统原生弹窗
7. 禁止修改原始文件名
8. 禁止在未确认前执行不可逆删除
9. 禁止把决策型提示做成底部胶囊
10. 禁止恢复媒体卡片右上角收藏图标
11. 禁止让主导航与自定义分组同时高亮
12. 禁止让小分组显示其他分组内容
13. 禁止交互区缺失 `WebkitAppRegion: no-drag`
14. 禁止 `tokens.css` 在 `@tailwind` 之后加载
15. 禁止在滚动中让已清晰显示的卡片再次降回低清层导致闪烁
