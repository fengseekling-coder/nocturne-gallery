# Visual Asset Checklist

This folder is reserved for public README screenshots and workflow GIFs. Do not add placeholder or fake generated product screenshots. Capture these assets from the real app when the first public alpha is ready.

## Required Assets

### `screenshot-gallery.png`

Show the main gallery view with a realistic local asset collection. It should communicate the masonry/grid browsing experience, selected group context, and dark visual style.

### `screenshot-inspector.png`

Show the inspector with a selected asset, editable prompt area, metadata, and attachment cards. The goal is to explain why this is more than a photo grid.

### `screenshot-preview-sync.png`

Show full-screen preview behavior with the inspector context synchronized to the active asset. Capture this only from the real app state.

### `demo-import.gif`

Show importing from local files, local folders, or a web-dragged asset. Include visible progress or feedback so the workflow does not look silent.

### `demo-drag-out.gif`

Show one or more selected assets being dragged out of Nocturne Gallery into another desktop context. Use a real workflow and avoid exposing private files.

### `social-preview.png`

GitHub social preview image. It should include the product name, one-line positioning, and a real or carefully composed app screenshot once available.

## README Preview Paths

The README reserves these first-phase paths:

- `docs/assets/screenshot-gallery.png`
- `docs/assets/screenshot-inspector.png`
- `docs/assets/demo-import.gif`

Until those files exist, README should not embed them as images. Use text-only placeholders to avoid broken Markdown previews.
